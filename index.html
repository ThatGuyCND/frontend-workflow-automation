“The more complex and self-sufficient the automated system, the more difficult it will be understand and operate.”

The Paradoxes and Ironies of Automation, David Wentzel.
http://www.davewentzel.com/content/paradoxes-and-ironies-automation
The key to a successful automated system is one that:
- Takes advantage of the natural interface presented by it’s component parts
- breaks down complexity into simplicity
- makes compromises intentionally, not circumstantially

- natural interface: A lot of tools already come with CLI integration - why are you using a GUI?
- complexity: Good complex tasks are comprised by many well-defined simple tasks.
- intentional compromise: simpler mental models instead of performance; less code, but less extensible

NodeJS is NOT server-side JS.

NodeJS is a stand-alone javascript runtime that is powered by V8, Chrome’s JavaScript engine
- NodeJS is a giant ecosystem of JS tooling
- NodeJS is a platform or an environment that let us run Javascript in our system/machine rather than the browser.
- Although it’s main purpose is to serve as a backend for web applications, developers quickly adopted it as a quick and easy solution for scripting.

Node is typically used in one of two ways:


  1. Build Utilities: things that help one build front-end applications
  2. Build Web Server: Backend server-side development
Web Server: think something like a JS SDK for testing a RAML spec of creating dummy REST end-points.
NPM is the package manager for NodeJS. It is bundled and installed automatically with the environment. NPM runs through the command line and allows users to install NodeJS applications, packages or utilities that are available on the NPM registry.

Bower Functions

bower search <name>

bower info <name>#<version> [<property>]

bower init

bower register <name> <url>

bower link

bower link <name>

- search: let's you search for a specific package using a string
- info: displays overall information of a package or of a particular version.
- init: interactively create a bower.json file
- register: register a package
- link: two step process. allows a local development version of a package to be symlinked globally. executing bower link <name> will create a link in the components directory of the previously created link.

Prod or Dev?


# install for production use.
$  bower install jquery --save
              


# Install for testing, debugging, development purposes only.
$  bower install modernizr --save-dev

- using the `save` flag when installing a dependency marks that dependency to always be available, which is typically reserved for production use.
- using the `save-dev` flag when installing a dependency is intended for development, debugging, and testing purposes only

gulp.src('src/main.mycss' )
  .pipe(stylus())
  .pipe(rename({ ext: 'css' }))
  .pipe(autoprefixer())
  .pipe(cssmin())
  .pipe(header( '/* All Right Reserved */' ))
  .pipe(gulp.dest( 'dist'));
- Gulp is written in JS and is stupid easy to use.
- If you’re familiar with using the pipe symbol in your CLI, then you’ll immediately get what Gulp is doing.
- Code over config
- Start with a _thing_ and do a whole bunch of other things to it.
- Gulp is truly asynchronous because it uses NodeJS streams.
- simple API
- `gulp.task`
- `gulp.src`
- `gulp.dest`
- `gulp.watch`
- less I/O operations as most piped streams are processed in memory.

The Gulp Task


  gulp.task(name [,dep],fn)
                

gulp.task(’minify',[‘jshint’],function() {
  return gulp.src('./src/**/*.js’)
    .pipe(concat(‘all.js’))
    .pipe(uglify())
    .pipe(gulp.dest(‘./build/’));
});
- name: name of the task we want to register
- dep(optional): task dependencies to run before this task
- fn: a callback function that returns a Gulp stream
